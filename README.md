# mksocket-journal

## 项目简介

mksocket-journal 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态 HTML 内容的存储与展示空间。

项目名称中的 "mksocket" 代表一种轻量级的页面构建方式，"journal" 则体现了其作为内容记录与归档的定位。

## 目录说明

```
mksocket-journal/
├── pages/           # 存放独立 HTML 页面文件
├── assets/          # 页面所需的静态资源（CSS、JS、图片等）
├── index.html       # 仓库首页/导航页
└── README.md        # 本文件
```

- `pages/`：核心目录，每个子文件夹或文件对应一个独立的 HTML 页面归档。
- `assets/`：存放页面共用的或单独引用的资源文件。
- `index.html`：可选的导航入口，用于浏览仓库内所有归档页面。

## 页面归档说明

- 每个归档页面均为独立的 HTML 文件，不依赖外部数据库或后端服务。
- 页面内容以静态形式呈现，可直接通过浏览器打开或部署到任何静态托管服务。
- 归档内容不包含任何具体的域名、网址或营销信息，仅作为资料记录。
- 页面命名建议采用有意义的英文或数字组合，便于识别与维护。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改归档页面。
- 如需提交页面归档，请遵循以下原则：
  - 页面应自包含，避免引用外部不可控资源。
  - 文件结构保持清晰，建议将相关页面放入 `pages/` 下独立子目录。
  - 避免在页面中包含敏感、违法或侵犯第三方权益的内容。
- 欢迎通过 Issue 提出改进建议，但请注意本仓库不承接任何商业合作或推广需求。
